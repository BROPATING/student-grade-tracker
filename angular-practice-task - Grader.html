Angular Practical - 1

Student Grade Tracker

Project Overview

Build a student grade tracking system where teachers can manage student information, add grades for different subjects, calculate averages, and view performance statistics with visual indicators.

Core Requirements

  1. Component Structure:
    • Create an AppComponent as the root
    • Create a StudentListComponent to display all students
    • Create a StudentCardComponent for individual student info
    • Create an AddStudentComponent for adding new students
    • Create a GradeFormComponent for adding grades to students
    • Create a StatsComponent to show class statistics
  2. Services:
    • Create a StudentService with providedIn: 'root'
    • Student model should include: id, name, email, enrollmentDate, grades array (subject, score, date)
    • Implement methods: getStudents(), addStudent(), deleteStudent(), addGrade(), calculateAverage(), getLetterGrade()
    • Create a GradeCalculatorService for grade calculations and statistics
  3. Data Binding:
    • Use [(ngModel)] for all form inputs (student name, email, subject, grade)
    • Use property binding for button states and form validation
    • Use event binding for form submissions and student actions
    • Use class binding to apply success/warning/danger classes based on grades
    • Use style binding for progress bars showing student performance
    • Use template reference variables for form validation feedback
  4. Directives:
    • Use *ngFor with trackBy to display students and their grades
    • Use *ngIf to show/hide sections (no students message, grade details, statistics)
    • Use *ngSwitch to display different status icons based on letter grades (A, B, C, D, F)
    • Use ngClass to apply conditional styling for grade levels
    • Use ngStyle for dynamic width of performance bars
  5. Custom Directive:
    • Create a GradeColorDirective that automatically colors grade text based on score
    • Use @Input() to receive the grade value
    • Use Renderer2 to apply colors (green for 90+, blue for 80-89, yellow for 70-79, orange for 60-69, red for below 60)
  6. Pipes:
    • Use date pipe for enrollment and grade dates
    • Use number pipe to format GPA/average with 2 decimal places
    • Create a custom LetterGradePipe that converts numeric grades to letter grades
    • Create a custom GpaCalculatorPipe that calculates GPA from grades
    • Create a custom FilterByPerformancePipe to filter students by grade range
  7. Component Communication:
    • Use @Input() to pass student data to child components
    • Use @Output() with EventEmitter for delete student, add grade events
    • Share data between components using services
  8. Lifecycle Hooks:
    • Use ngOnInit to load student data
    • Use ngOnChanges to recalculate averages when grades change
    • Use ngAfterViewInit if implementing any view-based calculations

Features to Implement

  • Add new students with name, email, and enrollment date
  • Display list of all students with their basic info and average grade
  • Add grades for different subjects to each student
  • Calculate and display overall average for each student
  • Convert numeric averages to letter grades (A, B, C, D, F)
  • Show all grades for a student with subject names and dates
  • Delete students from the system
  • Display class statistics (total students, class average, highest/lowest performer)
  • Visual indicators for performance levels (color-coded grades)
  • Filter students by performance (Excellent, Good, Average, Poor)
  • Edit student information
  • Delete individual grades
  • Add grade weighting system (assignments 40%, midterm 30%, final 30%)
  • Implement subject-wise class averages
  • Add a chart/graph visualization for grade distribution (using divs and CSS)
  • Create a student search feature
  • Add validation for grade ranges (0-100) with visual feedback
  • Implement a "Students at Risk" section showing students with averages below 70
  • Component structure and proper hierarchy (20%)
  • Service implementation with proper business logic (15%)
  • All binding types implemented correctly (15%)
  • Effective use of all required directives (15%)
  • Custom directive and pipes working correctly (15%)
  • Parent-child communication and data flow (10%)
  • Code quality and Angular best practices (10%)

General Guidelines for All Tasks

Technical Requirements

  • Use Angular CLI to generate all components, services, directives, and pipes
  • Follow Angular naming conventions and file structure
  • Import necessary modules (FormsModule for ngModel)
  • Use TypeScript strict mode and proper typing
  • Write clean, readable, and well-commented code

Styling

  • Add basic CSS styling to make your application presentable
  • Use component-specific styles in each component's CSS file
  • Experiment with ViewEncapsulation if desired
  • Make the UI user-friendly and intuitive

Testing Your Application

  • Ensure all features work as expected
  • Test with different data scenarios (empty states, multiple items, edge cases)
  • Verify that all directives and pipes work correctly
  • Check that component communication is functioning properly

Submission Requirements

  1. Create a new Angular project using ng new project-name
  2. Implement all required features and components
  3. Test thoroughly to ensure everything works
  4. Zip your project folder (exclude node_modules)
